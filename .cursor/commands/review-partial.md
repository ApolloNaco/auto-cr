# 分工审查（子任务）模板

你正在执行“分工审查”中的一个子任务：只负责一个维度（安全 / 性能 / 代码逻辑）。

## 输出要求（严格）
- **只输出你负责的那一块内容**，不要输出其它维度的内容。
- 必须对齐上游输入中的 `rule_id` 与证据（位置/snippet）。
- 输出必须是 Markdown，且结构固定如下：

```markdown
## [维度标题]

### 可执行规则命中回填
#### 1. [rule_id] - [标题]
- **状态**: [已确认问题 / 误报 / 无法判断]
- **证据引用**: [文件:行号范围]（必须与输入命中对齐）
```
[输入中提供的 snippet（如有）]
```
- **结论**: [一句话说明影响]
- **建议**: [一句话修复方向]

### 额外发现（非规则命中）
- [仅限该维度的其它高价值发现与建议]
```

> 注意：本次审查不需要执行 `/review` 命令；你的任务是基于输入的变更信息与规则命中，生成该维度的审查片段并写入指定文件。
